About This Project

This is a foundational web development assignment focused on building a well-structured HTML5 page. The primary goal is to use **semantic tags** to create a document that is both human-readable and machine-readable.

We are avoiding the use of non-semantic tags like <div> and <span> in favor of elements that describe their content, such as <header>, <nav>, <main>, and <footer>.

Our Core Principles

HTML5 Semantic Tags

Semantic HTML helps both browsers and developers understand the meaning and role of different parts of a web page. This improves readability and provides a clearer structure. For instance, using an <article> tag for a self-contained piece of content makes its purpose explicit.

Accessibility (A11y)

A web page is not complete if it isn't accessible to all users. By using proper heading hierarchies, descriptive text, and ARIA attributes (like the aria-label on our navigation), we ensure that people using assistive technologies like screen readers can easily navigate and understand the content.

Search Engine Optimization (SEO)

A well-structured page with logical headings and descriptive metadata is a core part of SEO. Search engine crawlers can efficiently index the page and understand its main topics, which helps the page rank higher in search results. This project uses a clear <title> and <meta> description tag to provide context.
